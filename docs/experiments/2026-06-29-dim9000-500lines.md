# 2026-06-29 Dim9000 500-Line Run

This run tested whether more TinyStories exposure improves prompt differentiation at 9000 dimensions.

## Command

```powershell
.\dzeta_train_smoke_native.exe `
  --corpus benchmarks\data\tinystories_sample.txt `
  --seconds 0 `
  --target-lines 500 `
  --oscillators 65536 `
  --dimensions 9000 `
  --tokens 24 `
  --progress-seconds 60 `
  --temperature 1.0 `
  --learning-rate 1.0 `
  --threads 20 `
  --parallel-min-dim 1 `
  --shuffle-lines `
  --update-probability 0.8 `
  --update-noise 0.001 `
  --random-init-scale 0.001 `
  --autosave-seconds 0 `
  --save-model C:\dzeta_backup\models\dim9000_500lines_stochastic.dzeta.bin
```

The model was saved only once at the end. This avoided the large I/O cost of minute-by-minute autosaves.

## Interpretation

This is not a simple template collapse. The outputs are grammatical, story-like, and richer than early low-data runs. However, prompt contrast is weaker than in the 144-line run. The model learned a stronger global TinyStories genre prior:

```text
beautiful / butterfly / adventure / favorite / something special / microscope or telescope /
happened suddenly / grandfather carefully climbed / together / everyone
```

The right interpretation is genre convergence:

- 144 lines produced sharper prompt-specific differences.
- 500 lines produced better grammar and vocabulary but weaker contrast.
- TinyStories is itself highly formulaic, so more data pushed the field toward a general children's-story attractor.
- Lower loss did not mean better prompt differentiation.

## Inspector Findings

Inspector summary:

```text

```

`robot` had token links, but mostly to high-frequency grammatical bridge words:

```text
robot -> and, was, to, They, the, "...", She, One, He, had, The, said, big, his, in, it, her, on, he, day
```

`assistant` and `intelligence` returned `no_links`, which is expected if those exact tokens were not learned from the TinyStories corpus. Prompt generation still works because prompts are projected through the field, but token-level saved links require learned oscillator entries.

`grandfather` showed a different diagnostic pattern: low next/context similarity but high transition similarity. That suggests it is acting more like a phase-transition motif inside the story attractor than a clean semantic neighbor.

## Next Experiment

The next useful experiment should not simply add more TinyStories lines. It should test whether DZETA can preserve high-dimensional sample efficiency while improving prompt contrast:

- use a more diverse mixed corpus: stories, instructions, dialogue, object descriptions, simple reasoning;
- compare temperature `0.5`, `0.7`, and `1.0` from the same saved model;
- inspect `robot`, `assistant`, `intelligence`, and domain-specific terms after training;
- add content-token weighting and frequency-attractor suppression;
- test the new resonance subword traces in a separate run.
